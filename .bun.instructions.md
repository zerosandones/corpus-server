# Bun and Elysia.js Best Practices Guide

This document defines architectural standards and best practices for implementing Corpus Server using the **Bun** runtime and **Elysia.js** framework. These rules emphasize sub-millisecond API performance, runtime type safety, and clean structuring.

---

## 1. Native Bun APIs
Bun features incredibly optimized, low-overhead native functions. **Always** prefer Bun's native equivalents over Node.js polyfills or standard external libraries.

### File Operations (`Bun.file` and `Bun.write`)
Do not use standard Node `fs` or `fs/promises`.
- `Bun.file(path)` returns a lightweight `BunFile` pointer without loading the file into memory until requested.
- `Bun.write(path, data)` is highly optimized for asynchronous disk writes.

```typescript
//  Reading document text
const doc = Bun.file('./storage/engineering/architecture.md');
const text = await doc.text(); // Loads and parses efficiently

//  Writing raw content to file
await Bun.write('./storage/engineering/architecture.md', '# New Content');
```

### In-Memory Database (`bun:sqlite`)
For search indexes and document metadata caching, use Bun's built-in, ultra-fast SQLite driver. Do not pull in heavy ORMs like Prisma or external drivers.

```typescript
import { Database } from "bun:sqlite";

// Initialize a ultra-fast in-memory FTS database
const db = new Database(":memory:");
db.run("CREATE VIRTUAL TABLE docs USING fts5(path, title, body);");
```

## 2. Elysia.js API Design Patterns
Elysia is an exceptionally fast, type-safe API framework built specifically for Bun. Implement routing, validation, and lifecycle management according to these patterns:

### Schema Validation with TypeBox
Elysia natively supports TypeBox schemas via the `t` object. **Every** route path, query parameter, and payload body must specify a TypeBox validation schema. This guarantees both compilation-level and runtime validation.

```typescript
import { Elysia, t } from 'elysia';

const app = new Elysia()
  .put('/documents/*', async ({ params, body, headers }) => {
    const documentPath = params['*'];
    // Logic to save document...
    return { success: true };
  }, {
    params: t.Object({
      '*': t.String({ description: "Relative file system path" })
    }),
    body: t.Object({
      frontmatter: t.Object({
        title: t.String(),
        description: t.String(),
        tags: t.Optional(t.Array(t.String()))
      }),
      body: t.String()
    })
  });
```

### Context & State Decoration (`decorate` and `derive`)
- Use `decorate` to inject persistent, startup-configured singleton resources (such as the parsed dynamic `config` object, in-memory SQLite database, or filesystem helper clients).
- Use `derive` to parse request-level parameters (such as extracting token properties via `Authorization: Bearer` and loading user context dynamically).

```typescript
import { Elysia } from 'elysia';
import { Database } from 'bun:sqlite';
import { parseConfig } from './config-loader.ts';

const config = parseConfig('./config.yaml');

const app = new Elysia()
  .decorate('config', config)
  .decorate('db', new Database(config.database.path))
  .derive(({ headers, config }) => {
    // Access injected global configuration properties inside request scope
    const authHeader = headers['authorization'];
    const token = authHeader?.split(' ')[1];
    
    // Auth type verification dynamically driven by configuration
    const user = (config.auth.type === 'static' && token) 
      ? lookupTokenInMemory(token, config.auth.tokens) 
      : null;
      
    return { user };
  });
```

## 3. High Performance Testing (`bun:test`)
Use Bun's built-in high-performance testing environment instead of Jest, Vitest, or Mocha.

- Use standard `describe`, `it`/`test`, and `expect` syntax.
- Mock external files or system dates using native `mock` utilities.
- Elysia features a native `.handle()` method, allowing you to execute end-to-end integration tests in-memory without starting a real TCP port, increasing testing speeds exponentially.

```typescript
import { describe, expect, it } from "bun:test";
import { app } from "../src/index.ts";

describe("GET /documents", () => {
  it("should return raw Markdown when requested", async () => {
    const response = await app.handle(
      new Request("http://localhost/engineering/architecture", {
        headers: { "Accept": "text/markdown" }
      })
    );
    expect(response.status).toBe(200);
    expect(response.headers.get("Content-Type")).toContain("text/markdown");
  });
});
```

## 4. Performance Checklist
- **Avoid Async Synchronous Blocks**: Never block Bun's main event loop. Always await heavy operations (`await doc.text()`, search loops, etc.).
- **Cache Frontmatter**: Parsing YAML frontmatter repeatedly is expensive. Keep an in-memory Cache (e.g. SQLite database tables) that updates in response to write requests or disk watchers (`chokidar` or native Bun file system events).
- **TypeScript Transpilation**: Bun runs TypeScript directly natively, so do not compile `.ts` to `.js` files for running. Simply use `bun run src/index.ts`.
