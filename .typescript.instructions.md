# TypeScript Best Practices Guide

This document establishes the TypeScript coding standards for the Corpus Server codebase. These rules are designed to ensure excellent type safety, optimize developer productivity, and facilitate seamless parsing and editing by AI agents.

---

## 1. Type Safety & Configuration
- **Strict Mode**: `strict: true` must be enabled in `tsconfig.json`.
- **No Implicit Any**: Never allow implicit `any`. If a type is truly unknown, explicitly use `unknown` rather than `any` to force type assertions downstream.
- **Strict Null Checks**: Explicitly declare union types with `null` or `undefined` (e.g., `string | null`) to prevent runtime null-reference exceptions.

```typescript
// ❌ Avoid
function processMetadata(data: any) {
  console.log(data.title.toLowerCase()); // Unsafe
}

//  Recommend
interface DocumentMeta {
  title: string;
  description?: string;
}

function processMetadata(data: unknown) {
  if (data && typeof data === 'object' && 'title' in data) {
    const meta = data as DocumentMeta;
    console.log(meta.title.toLowerCase()); // Safe
  }
}
```

---

## 2. Types vs. Interfaces
- **Use `interface`** for structural definitions of object shapes, standard API payloads, and classes. Interfaces are extendable and mergeable.
- **Use `type`** for unions, intersections, utility helpers, function signatures, and mapping primitives.

```typescript
//  Use Interface for objects
interface UserContext {
  sub: string;
  type: 'human' | 'agent';
  roles: string[];
}

//  Use Type for unions/utilities
type DocumentPermission = 'read' | 'write' | 'none';
type CallbackFunction = (err: Error | null, result?: boolean) => void;
```

---

## 3. Avoid Enums (Prefer Const Assertions & Literal Unions)
Standard TypeScript `enum` compilation generates runtime overhead and confusing bidirectional lookup objects. Prefer literal unions or `const` assertions.

```typescript
// ❌ Avoid
enum SecurityLevel {
  Public = "public",
  Private = "private",
  Confidential = "confidential"
}

//  Recommend
export const SecurityLevel = {
  Public: "public",
  Private: "private",
  Confidential: "confidential"
} as const;

export type SecurityLevelType = typeof SecurityLevel[keyof typeof SecurityLevel]; // "public" | "private" | "confidential"
```

---

## 4. Import & Export Conventions
- **Type-Only Imports**: Use `import type` for purely structural symbols, interfaces, or types. This reduces transpilation output and prevents runtime circular dependency bugs.
- **Barrel Imports**: Avoid complex, recursive index file imports (barrel imports) that bloat loading speeds and break tree-shaking. Prefer importing modules directly from their source files.

```typescript
//  Recommend type-only imports
import type { UserContext } from './types.ts';
import { parseFrontmatter } from './parser.ts';
```

---

## 5. Asynchronous Operations
- **Prefer Async/Await**: Always prefer `async/await` syntax over raw `.then().catch()` chaining for readability and consistent trace logging.
- **Typed Catch Clauses**: When catching errors, remember the error variable in JS/TS is `unknown` by default. Type-guard it before using.

```typescript
//  Recommend
async function readDocumentContent(path: string): Promise<string> {
  try {
    const file = Bun.file(path);
    return await file.text();
  } catch (error) {
    if (error instanceof Error) {
      throw new Error(`Failed to read document at ${path}: ${error.message}`);
    }
    throw new Error('An unknown file system error occurred.');
  }
}
```

---

## 6. Nullish Operations & Optional Chaining
- Use Optional Chaining (`?.`) to safely fetch nested values.
- Use Nullish Coalescing (`??`) to supply default values rather than logical OR (`||`), to prevent hiding valid falsy values like `""` (empty string) or `0`.

```typescript
// ❌ Avoid (hides empty titles)
const title = doc.title || "Untitled";

//  Recommend
const title = doc.title ?? "Untitled";
const nestedTags = doc.metadata?.tags ?? [];
```
